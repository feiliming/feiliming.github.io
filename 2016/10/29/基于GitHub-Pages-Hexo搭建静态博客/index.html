基于GitHub Pages + Hexo搭建静态博客

[TOC]

工具/说明

1.Windows 7 操作系统

2.Node.js环境

Node.js
大家都知道JavaScript是在客户端浏览器中运行的，而Node.js是让JavaScript脱离浏览器运行在服务器端的平台，使用它可以进行服务器端编程，它遵循CommonJs规范，使用Google Chrome V8的JavaScript引擎，你能想到的它都能干，你想不到的它还能干。

感兴趣可以看一下各种Node.js开发指南。

npm
npm 官方 doc
npm，node package manager，JavaScript的包管理器，它是一个命令行工具，
npm 使用 package.json 来存储项目的元数据，最好通过配置文件 package.json 来管理项目的依赖包，类似maven管理项目的pom.xml，

npm is the package manager for JavaScript. Find, share, and reuse packages of code from hundreds of thousands of developers — and assemble them in powerful new ways.

关于npm你只要会下面几个命令就够用了：

  • npm -v，查看npm版本
  • npm -h，查看帮助

  • npm update npm -g，更新npm

  • npm install webapck ，本地安装webpack

  • npm install webpack -g ，全局安装webpack
  • npm install webpack --save-dev，本地安装webpack，并将依赖关系写入到package.json的devDependencies开发依赖中
  • npm install webpack --save，本地安装webpack，并将依赖关系写入到package.json的dependencies生产依赖中
  • npm update webpack，更新webpack
  • npm uninstall webpack，卸载webpack

本地安装就是切换到项目所在目录进行安装，安装后项目根目录会出现node_modules，里面是当前项目安装的包及依赖。
全局安装会安装到Node.js的安装目录下，然后所有项目都可以使用。

推荐能本地安装的都本地安装，即使必须全局安装的最好也在本地安装一份，这样能体现出项目的依赖关系。

如果下载慢指定淘宝镜像，--registry = http://registry.npm.taobao.org。

4.Git

Git，一个开源免费的分布式版本控制系统。
本文使用的是 TortoiseGit。 其他更多GUI Clients。

注意，别跟GitHub混淆，GitHub是一个项目托管平台，它的版本控制是基于Git实现的。

5.GitHub Pages

GitHub Pages，它可以创建用户、组织或者项目的站点，站点是静态的，但它是免费的。一个GitHub账号可以创建一个用户站点，项目站点不受限制，每个项目都可以有一个站点。

本文说明怎么创建用户站点，组织站点和用户站点是一样的。

6.Hexo

Hexo是一个快速、简单且功能强大的博客框架。Hexo使用Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用漂亮的主题生成静态网页。

方法/步骤

1.安装Node.js + npm

下载最新的Nodejs，本文使用的是v6.2.0，一路下一步安装，安装Node.js时会自带npm，自带的版本是3.8.9。

如果你安装的Node.js是稳定版v4.4.5，自带的npm版本是v2.15.5，建议升级到最新版本，npm update npm -g。

2.安装Git

安装 TortoiseGit。

3.安装Hexo

  • 打开Git Bash，全局安装hexo-cli
    npm install -g hexo-cli
  • 切换到工作目录，本文是F盘的gh_blog
    cd f:/gh_blog
  • 执行hexo命令，会在ghb1文件夹下新建所需要的文件，npm install会安装package.json中的包
    hexo init ghb1
    cd ghb1
    npm install
  • 此时ghb1目录下文件目录如下：

node_modules
项目已安装的依赖包

scaffolds
放模板的文件夹，文章模板

source
资源文件夹，创建的文章会在这里，解析之后会放到public文件夹

themes
主题文件夹，landscape是默认主题，还可以安装很多主题，hexo会根据主题生成静态页面

_config.yml
博客站点的配置文件，很多参数都在_config.yml配置，包括你安装的插件

package.json
项目依赖信息管理文件，打开它，可以看见已经默认安装了一些插件，比如：EJS（已经被DoneJS替代）、Stylus、Markdown

4.配置_config.yml

配置标题、子标题

还有很多可配置的参数，自行参考docs

5.启动服务

hexo server
根据提示访问http://localhost:4000
默认有一篇文章，Hello World

6.新建一篇文章

hexo new post '基于GitHub Pages + Hexo搭建静态博客'
post为模板名称，如果不指定则使用_config.yml中配置的default_layout，如果标题有空格，需要使用引号括起来。

7.编辑文章内容

新建文章后会在/source/_posts/目录下生成对应的*.md文件，.md是使用Markdown语法编写的文件，文本编辑器打开.md文件，使用Markdown语法写一些内容。
